## اختبر بنفسك الآن 🚀

### الخطوة 1: إنشاء رسالة تجريبية

```bash
# عبر الواجهة الرسومية:
1. اذهب إلى http://localhost:3000/dashboard/messages/new
2. ملأ التفاصيل:
   - Title: "رسالة تجريبية"
   - Content: "مرحبا بك"
   - Trigger Type: DATE (أو SWITCH أو KEYHOLDER)
   - Scheduled Time: الآن أو وقت قريب
   - Add Recipients: بريدك الحقيقي أو هاتف

3. انقر Save
```

### الخطوة 2: اختبر الإرسال

```bash
# الطريقة الأسهل - عبر الواجهة التفاعلية:
1. اذهب إلى http://localhost:3000/dashboard/test
2. انقر "تحميل الرسائل" 
3. اختر رسالتك من القائمة
4. (اختياري) اختر قناة محددة للاختبار
5. انقر "أرسل الآن"
6. شاهد النتائج مباشرة ✅
```

### الخطوة 3: تحقق من البريد

```
إذا أضفت بريد حقيقي:
- تحقق من البريد الوارد (inbox)
- تحقق من مجلد Spam إذا لم تجده
- يجب أن تصل بعد ثوانٍ قليلة
```

## سيناريوهات الاختبار

### سيناريو 1: رسالة فورية (EMAIL)

**الهدف:** التأكد من أن البريد الإلكتروني يصل فوراً

```
1. إنشاء رسالة:
   - Type: DATE
   - Scheduled: الآن
   - Channel: EMAIL
   - Recipient: بريدك أو بريد تجريبي

2. الاختبار:
   - اذهب إلى /dashboard/test
   - Load → Select → Send Now
   
3. النتيجة المتوقعة:
   ✅ deliveryStatus.channel = "EMAIL"
   ✅ deliveryStatus.status = "SENT"
   ✅ البريد يصل في الثواني القليلة الأولى
```

### سيناريو 2: رسالة مجدولة (مستقبلية)

**الهدف:** التأكد من أن الجدولة تعمل للأوقات المستقبلية

```
1. إنشاء رسالة:
   - Type: DATE
   - Scheduled: غداً الساعة 14:30
   - Channel: EMAIL
   - Recipient: بريدك

2. الانتظار:
   - يجب أن تُرسل تلقائياً غداً الساعة 14:30 UTC
   - يحدث عبر Cron Job يومياً

3. للتسريع - قم بتشغيل المعالج يدويا:
   curl http://localhost:3000/api/cron/process-switches
```

### سيناريو 3: رسالة SWITCH مع جدولة

**الهدف:** التأكد من أن SWITCH يدعم الجدولة

```
1. إنشاء رسالة:
   - Type: SWITCH (جديد!)
   - Scheduled: الآن أو وقت قريب
   - Channel: EMAIL + SMS + WhatsApp
   - Recipient: معلومات شاملة

2. الاختبار:
   - /dashboard/test → Load → Select → Send
   
3. النتيجة المتوقعة:
   ✅ EMAIL ينجح
   ✅ SMS معد (مُرسل للاختبار)
   ✅ WhatsApp معد (مُرسل للاختبار)
```

### سيناريو 4: عدة مستقبلين وقنوات

**الهدف:** التأكد من معالجة عدة مستقبلين وقنوات

```
1. إنشاء رسالة:
   - Type: كل نوع
   - Recipient 1: أحمد - EMAIL ahme@example.com
   - Recipient 2: فاطمة - SMS +966123456789
   - Recipient 3: علي - WhatsApp +966987654321

2. الاختبار:
   - /dashboard/test → Send

3. النتيجة المتوقعة:
   ✅ 3 مستقبلين معالجون
   ✅ 3 قنوات مختلفة
   ✅ deliveryStatus يعرض حالة كل قناة
```

### سيناريو 5: رسالة KEYHOLDER مع جدولة

**الهدف:** التأكد من أن KEYHOLDER يدعم الجدولة

```
1. إعداد Keyholder:
   - Settings → Add Keyholder
   - أضف بريد/هاتف الشاهد الموثوق

2. إنشاء رسالة:
   - Type: KEYHOLDER (جديد!)
   - Scheduled: وقت معين
   - اختر Keyholder الذي أضفته

3. الاختبار:
   - /dashboard/test → Send

4. النتيجة المتوقعة:
   ✅ رسالة تُرسل للشاهد الموثوق
   ✅ في الوقت المحدد
```

## إذا حدثت مشاكل

### المشكلة: الرسالة لا تظهر في /dashboard/test

```
السبب المحتمل: الرسالة ليست ACTIVE
الحل:
1. تأكد أنك نقرت SAVE وليس DRAFT
2. في API: تحقق من status = 'ACTIVE'
3. جرّب إنشاء رسالة جديدة
```

### المشكلة: البريد لا يصل

```
السبب المحتمل: RESEND_API_KEY ليس صحيحاً
الحل:
1. تحقق من .env.local:
   RESEND_API_KEY=re_xxx (يجب أن يبدأ بـ re_)
2. تحقق من EMAIL_FROM (بريد حقيقي)
3. تحقق من console logs للأخطاء

السبب الآخر: بريد الإستقبال غير صحيح
الحل:
1. استخدم بريدك الحقيقي للاختبار
2. تحقق من Spam folder
3. أضف البريد الموثوق في الإعدادات
```

### المشكلة: SMS/WhatsApp لا تعمل بالكامل

```
هذا متوقع! ✅
SMS و WhatsApp معدة للتكامل فقط حالياً
الحالة:
- ✅ المستقبلون يُحفظون
- ✅ الرسائل تُرسل (معاملة كمرسلة)
- 🔄 التكامل الحقيقي: يحتاج Twilio/AWS

للاستخدام الفعلي:
1. اشترك في Twilio أو AWS
2. أضف credentials في .env
3. حدّث triggerMessages() في switch-engine.ts
```

### المشكلة: جدولة لا تعمل

```
السبب المحتمل: Cron job لم تعمل بعد
الحل:
1. للاختبار الفوري: اذهب إلى /dashboard/test
2. لاختبار Cron: 
   curl http://localhost:3000/api/cron/process-switches
3. تحقق من الوقت:
   - scheduledAt يجب أن يكون <= الوقت الحالي
   - الوقت بصيغة ISO: 2024-01-15T14:30:00Z
```

## نقاط مهمة تذكّر

### توقيت Cron
```
- يعمل يومياً في 8:00 AM UTC
- يعالج جميع الرسائل المستحقة
- في الإنتاج: عبر Vercel Cron
- للاختبار: استدعاء يدوي أو /dashboard/test
```

### حالات الرسائل
```
DRAFT → ACTIVE → SENT
- DRAFT: قيد الكتابة، لن تُرسل
- ACTIVE: جاهزة للإرسال
- SENT: تم الإرسال (واحد أو أكثر مستقبل)
```

### حالات المستقبلين
```
PENDING → SENT → DELIVERED (أو FAILED)
- PENDING: ينتظر الإرسال
- SENT: البريد أُرسل من الخادم
- DELIVERED: تأكيد وصول (EMAIL فقط حالياً)
- FAILED: فشل الإرسال
```
